---
description: Create a git commit with meaningful conventional commit message
---

Create a git commit following these steps:
1. Check git status to see all untracked and modified files
2. Stage all changes with git add .
3. Review the git diff of staged changes
4. Analyze the changes and draft a commit message using the format: `<type>: <description>`
   - Type must be one of: fix, chore, feat, or docs
   - Description should be clear and concise (lowercase, no period at end)
5. Create the commit with the message
6. Confirm the commit was successful
